# Лабораторная работа №4: Ввод-вывод и работа с файлами в языке C

## Описание работы
Эта лабораторная работа посвящена изучению работы с файлами в языке программирования C. В ходе выполнения рассматриваются следующие темы:
- Открытие и закрытие файлов с помощью `fopen` и `fclose`.
- Режимы открытия файлов (текстовые и бинарные).
- Посимвольный ввод-вывод с помощью `putc` и `getc`.
- Построчный ввод-вывод с помощью `fputs` и `fgets`.
- Форматированный ввод-вывод с помощью `fprintf` и `fscanf`.
- Работа с бинарными файлами с помощью `fread` и `fwrite`.
- Позиционирование в файле с помощью `fseek` и `ftell`.
- Проверка конца файла (`feof`) и ошибок (`ferror`).
- Создание простой файловой базы данных со структурами.

## Технологии и библиотеки
- **Язык программирования**: C.
- **Компилятор**: `gcc` (GNU Compiler Collection).
- **Используемые библиотеки**:
  - `stdio.h` — для работы с файлами (функции `fopen`, `fclose`, `putc`, `getc`, `fputs`, `fgets`, `fprintf`, `fscanf`, `fread`, `fwrite`, `fseek`, `ftell`, `feof`, `ferror`).
  - `stdlib.h` — для вспомогательных операций.

## Задания
1.  **Открытие и закрытие файла**: Открытие файла "test.txt" в режиме записи, проверка успешности открытия, вывод сообщения об ошибке при неудаче, закрытие файла.
2.  **Посимвольная запись и чтение**: Запись нескольких символов в файл с помощью `putc`, закрытие файла, открытие для чтения, посимвольное чтение с помощью `getc`, вывод содержимого на экран.
3.  **Работа со строками**: Запись нескольких строк в файл с помощью `fputs`, открытие для чтения, чтение строк с помощью `fgets`, вывод на экран.
4.  **Форматированный ввод-вывод**: Запись в файл чисел и строк с помощью `fprintf`, закрытие, открытие для чтения, чтение данных с помощью `fscanf`, вывод считанных значений.
5.  **Ввод с клавиатуры и запись в файл**: Чтение строки с клавиатуры с помощью `fgets`, запись её в файл, вывод содержимого файла на экран.
6.  **Бинарные файлы (простые данные)**: Создание массива целых чисел, запись его в бинарный файл с помощью `fwrite`, чтение обратно с помощью `fread`, вывод данных.
7.  **Работа со структурами в бинарных файлах**: Создание структуры `Person` (имя, возраст), создание одной структуры, запись её в файл, чтение обратно, вывод данных.
8.  **Массив структур**: Создание массива структур `Person`, запись массива в бинарный файл, чтение массива обратно, вывод всех элементов.
9.  **Позиционирование в файле**: Открытие файла, перемещение к заданной позиции с помощью `fseek`, чтение данных начиная с этой позиции, вывод результата.
10. **Размер файла**: Открытие файла, определение размера файла с помощью `fseek` и `ftell`, вывод размера.
11. **Проверка конца файла и ошибок**: Чтение файла до конца, проверка достижения конца файла с помощью `feof`, проверка наличия ошибок с помощью `ferror`.
12. **Простая файловая база людей**: Реализация базы данных со структурой `Person` (имя, возраст). Ввод данных с клавиатуры, запись в бинарный файл, чтение всех записей из файла, вывод на экран. Использование `fread` и `fwrite`, проверка открытия файла, закрытие файла после работы.

## Ответы на контрольные вопросы

**1. Что такое файл в языке C?**

В языке C файл — это последовательность байтов, с которой программа работает через поток. Файл может храниться на диске или другом носителе. Программа взаимодействует с файлом через указатель типа `FILE*`, который предоставляет интерфейс для чтения и записи данных. Файлы бывают текстовыми (содержат читаемые символы) и бинарными (содержат произвольные байты).

**2. Что такое FILE\*?**

`FILE*` — это указатель на структуру типа `FILE`, которая определена в библиотеке `stdio.h`. Эта структура содержит информацию о файловом потоке: текущую позицию чтения/записи, буфер, индикаторы ошибок и конца файла, режим открытия и т.д. Программист не работает с полями этой структуры напрямую, а передаёт указатель функциям работы с файлами (`fprintf`, `fread` и т.д.).

**3. Какие режимы открытия файла существуют?**

Основные режимы открытия файла:
- `"r"` — чтение (файл должен существовать).
- `"w"` — запись (создаёт новый файл или перезаписывает существующий).
- `"a"` — дозапись в конец файла (создаёт файл, если его нет).
- `"r+"` — чтение и запись (файл должен существовать).
- `"w+"` — чтение и запись (создаёт новый или перезаписывает).
- `"a+"` — чтение и дозапись в конец.
- `"rb"`, `"wb"`, `"ab"` — аналогичные режимы для бинарных файлов.

**4. Чем отличается текстовый файл от бинарного?**

Текстовый файл содержит данные в виде символов, разбитых на строки. При чтении/записи может происходить преобразование символов (например, перевод строки `\n` может преобразовываться в пару `\r\n` на Windows). Бинарный файл содержит данные в том виде, в котором они хранятся в памяти. Преобразований не происходит. Бинарные файлы обычно компактнее и позволяют напрямую записывать структуры и массивы.

**5. Что делают функции putc и getc?**

`putc` записывает один символ в файл. Принимает символ (int) и указатель на файл. Возвращает записанный символ или `EOF` при ошибке. `getc` читает один символ из файла. Принимает указатель на файл. Возвращает прочитанный символ или `EOF` при достижении конца файла или ошибке. Они работают с текстовыми и бинарными файлами.

**6. Что делают fgets и fputs?**

`fputs` записывает строку в файл. Принимает строку (char*) и указатель на файл. Не добавляет автоматически символ перевода строки. Возвращает неотрицательное число при успехе или `EOF` при ошибке. `fgets` читает строку из файла. Принимает буфер, максимальный размер и указатель на файл. Читает до символа перевода строки или до достижения лимита. Сохраняет символ перевода строки в буфер. Возвращает указатель на буфер или `NULL` при ошибке/конце файла.

**7. Что делают fprintf и fscanf?**

`fprintf` выполняет форматированную запись в файл. Работает аналогично `printf`, но выводит данные в файл. Принимает указатель на файл, форматную строку и аргументы. `fscanf` выполняет форматированное чтение из файла. Работает аналогично `scanf`, но читает данные из файла. Принимает указатель на файл, форматную строку и указатели на переменные. Возвращает количество успешно считанных элементов.

**8. Что делают fread и fwrite?**

`fread` читает блок данных из бинарного файла. Принимает указатель на буфер, размер одного элемента, количество элементов и указатель на файл. Возвращает количество успешно прочитанных элементов. `fwrite` записывает блок данных в бинарный файл. Принимает указатель на данные, размер одного элемента, количество элементов и указатель на файл. Возвращает количество успешно записанных элементов. Эти функции идеально подходят для работы с массивами и структурами.

**9. Что делают fseek и ftell?**

`fseek` перемещает указатель позиции в файле. Принимает указатель на файл, смещение (long) и исходную позицию (`SEEK_SET` — начало, `SEEK_CUR` — текущая позиция, `SEEK_END` — конец). Возвращает 0 при успехе или ненулевое значение при ошибке. `ftell` возвращает текущую позицию указателя в файле (количество байт от начала). Принимает указатель на файл. Возвращает текущую позицию (long) или -1L при ошибке.

**10. Что делают feof и ferror?**

`feof` проверяет, достигнут ли конец файла. Принимает указатель на файл. Возвращает ненулевое значение, если конец файла достигнут, и 0 в противном случае. `ferror` проверяет, произошла ли ошибка при работе с файлом. Принимает указатель на файл. Возвращает ненулевое значение, если ошибка была, и 0 в противном случае. Обычно используются после операций чтения/записи.

**11. Почему важно закрывать файл?**

Закрытие файла с помощью `fclose` важно по нескольким причинам. Во-первых, данные, записанные в файл, могут находиться в буфере, и `fclose` принудительно сбрасывает буфер на диск. Во-вторых, операционная система имеет ограничение на количество одновременно открытых файлов. В-третьих, это освобождает ресурсы, связанные с файловым потоком. Если не закрыть файл, возможна потеря данных и утечка системных ресурсов.

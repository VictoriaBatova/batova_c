# Лабораторная работа №3: Динамические массивы в языке C

## Описание работы
Эта лабораторная работа посвящена изучению динамического выделения памяти в языке программирования C. В ходе выполнения рассматриваются следующие темы:
- Выделение памяти с помощью функций `malloc`, `calloc`, `realloc`.
- Освобождение памяти с помощью функции `free`.
- Создание и использование одномерных и двумерных динамических массивов.
- Изменение размера массива во время выполнения программы.
- Операции добавления, удаления и вставки элементов в динамический массив.
- Создание динамических массивов структур.
- Предотвращение утечек памяти и ошибок с висячими указателями.

## Технологии и библиотеки
- **Язык программирования**: C.
- **Компилятор**: `gcc` (GNU Compiler Collection).
- **Используемые библиотеки**:
  - `stdio.h` — для ввода/вывода данных (функции `printf`, `scanf`).
  - `stdlib.h` — для работы с динамической памятью (функции `malloc`, `calloc`, `realloc`, `free`).

## Задания
1.  **Создание динамического массива**: Запрос у пользователя размера массива, выделение памяти с помощью `malloc`, заполнение массива с клавиатуры и вывод на экран.
2.  **Использование calloc**: Выделение массива с помощью `calloc` и демонстрация того, что все элементы инициализированы нулями.
3.  **Поиск элементов**: Создание динамического массива и реализация функций поиска максимального элемента, минимального элемента и суммы всех элементов.
4.  **Изменение размера массива (realloc)**: Создание массива из n элементов, увеличение размера до m элементов с помощью `realloc`, ввод новых элементов, вывод обновленного массива, затем уменьшение размера.
5.  **Удаление элемента**: Создание динамического массива, удаление элемента по индексу со сдвигом оставшихся элементов и уменьшением размера через `realloc`.
6.  **Вставка элемента**: Создание динамического массива, вставка элемента в заданную позицию с увеличением размера и сдвигом элементов.
7.  **Массив структур (динамический)**: Создание структуры `Student` (имя, возраст), выделение динамического массива структур, заполнение данными, вывод и поиск самого старшего студента.
8.  **Динамический список чисел**: Реализация полноценного динамического списка с функционалом:
    - Добавление элемента
    - Удаление элемента
    - Вывод списка
    - Изменение размера массива
    - Меню для выбора действий (цикл с выбором операций)

## Ответы на контрольные вопросы

**1. Что делает malloc?**

malloc (memory allocation) выделяет блок памяти заданного размера в байтах. Функция принимает один аргумент — количество байт для выделения. Возвращает указатель на начало выделенного блока. Выделенная память не инициализируется, поэтому в ней содержатся случайные значения (мусор). Если выделить память не удалось, функция возвращает NULL.

**2. Чем отличается malloc от calloc?**

Основных отличий три. Во-первых, malloc принимает один аргумент — размер в байтах, а calloc принимает два аргумента — количество элементов и размер одного элемента. Во-вторых, malloc не инициализирует память, а calloc заполняет всю выделенную память нулями. В-третьих, calloc работает немного медленнее из-за необходимости обнуления памяти.

**3. Что делает realloc?**

realloc (reallocation) изменяет размер ранее выделенного блока памяти. Функция принимает указатель на существующий блок и новый размер в байтах. При увеличении размера старые данные сохраняются, новые элементы не инициализируются. При уменьшении размера лишние данные отбрасываются. Адрес блока может измениться, поэтому результат нужно присваивать обратно в указатель. В случае ошибки возвращает NULL, а старый блок остается неизменным.

**4. Что произойдет при утечке памяти?**

При утечке памяти программа продолжает удерживать выделенную память, которая уже не нужна и на которую нет ссылок. Если утечка происходит многократно (например, в цикле), программа постепенно消耗 всю доступную память. Это может привести к замедлению работы системы, а затем к аварийному завершению программы из-за нехватки памяти. В долго работающих программах (серверы, системы реального времени) утечки особенно критичны.

**5. Зачем нужен free?**

free освобождает ранее выделенную динамическую память и возвращает её системе для повторного использования. Без вызова free память останется занятой до завершения программы. Это приводит к утечкам памяти и неэффективному использованию ресурсов. После вызова free указатель становится "висячим" — его желательно обнулить, чтобы случайно не обратиться к освобожденной памяти.

**6. Что произойдет при двойном освобождении памяти?**

Двойное освобождение (double free) — это когда free вызывается дважды для одного и того же указателя. Это приводит к неопределенному поведению. Программа может аварийно завершиться, данные могут повредиться, или ошибка может проявиться позже совершенно в другом месте. Это одна из самых опасных ошибок при работе с динамической памятью, так как ее последствия трудно предсказать и отладить.

**7. Как проверить, что malloc вернул NULL?**

Нужно проверить возвращенное значение на равенство NULL сразу после вызова функции:

```c
int *arr = malloc(n * sizeof(int));
if (arr == NULL) {
    printf("Ошибка: не удалось выделить память\n");
    exit(1);  // или предпринять другие действия
}
```

**8. В чем отличие статического и динамического массива?**

Статический массив создается на стеке или в статической памяти. Его размер задается на этапе компиляции и не может быть изменен во время выполнения. Память выделяется автоматически при входе в блок и освобождается при выходе. Динамический массив создается в куче (heap) с помощью malloc/calloc. Его размер может быть любым и определяться во время выполнения. Размер можно изменять с помощью realloc. Однако программист обязан вручную освобождать память с помощью free, иначе возникнут утечки.
